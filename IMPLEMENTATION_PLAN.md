## 2. テストプレイ結果（役割別の主な指摘）

### ⚙️ フロントエンドエンジニア
- **F-1 毎フレームの DOM 書き込み**: `loop()` が毎フレーム `updateHud()` / `updateNotice()` を呼び、`updateNotice()` は `innerHTML` を 60fps で再代入。値が変わらなくても再パース・再レイアウトが走る。
- **F-2 ターゲット選定の O(n²) ソート**: `pickTarget()` の `dense`（💣既定）/ `line`（⚡既定）が、ソート比較関数内で `densityScore()` / `lineScore()`（各 O(n)）を毎回呼ぶ。武器選択中は `render()` 経由で毎フレーム走り、敵が多い終盤＋2倍速でフレーム落ちの要因になりうる。
- **F-3 共有プレビュー用 meta 不足**: 共有機能があるのに `og:*` / `description` が無く、リンク共有時の見栄えが弱い。`theme-color` 等のモバイル最適化 meta も未設定。

### 🧭 UXデザイナー
- **U-1 ルール説明が `alert()`**: ネイティブダイアログはモバイルで URL が出る・主スレッドを止める・図鑑モーダルと見た目が不一致。
- **U-2 2段階設置の発見性**: 武器選択 → シート閉じ → 同じマス再タップ、の「再タップで設置」が盤面の「設置予定」表示頼みで、初見だと気づきにくい。
- **U-3 アイコンボタンのラベル**: ⏸ / 速度ボタンに支援テキスト（aria-label）が無い。

### 🎨 UIデザイナー
- **D-1 モーダルの一貫性**: 図鑑は洗練されたモーダルなのにルールだけ素の alert で体験が割れる（U-1と同根）。
- **D-2 favicon / アプリ表示名なし**: タブ・ホーム追加時のアイコンや表示名が未設定。

### 🔍 QAテスター
- **Q-1 アニメーション配慮**: `prefers-reduced-motion` 未対応（危険ピルの点滅などが常時動く）。
- **Q-2 構文・整合性**: `node --vm` による構文チェックは PASS（既存バグの混入なし）。回帰防止のため改修後も再チェックする。

## 3. 課題の擦り合わせ（採用 / 見送り）

| ID | 課題 | 各ロールの結論 | 判定 |
| --- | --- | --- | --- |
| F-1 | 毎フレーム DOM 書き込み | 値が変化したときだけ書き込めば挙動不変で軽くなる | **採用** |
| F-2 | O(n²) ターゲットソート | スコアを事前計算（decorate-sort）すれば順序維持で軽量化 | **採用** |
| F-3 | meta 不足 | 共有導線があるので費用対効果大・低リスク | **採用** |
| U-1/D-1 | ルールを alert→モーダル | 既存 `guideModal` 系 CSS を再利用し一貫性確保 | **採用** |
| U-2 | 2段階設置の案内 | 武器選択時にトースト1行で補助（盤面表示は維持） | **採用** |
| U-3 | aria-label | 1行追加で改善 | **採用** |
| Q-1 | reduced-motion | CSS メディアクエリのみ・低リスク | **採用** |
| D-2 | favicon/表示名 | data URI の絵文字 favicon ＋ アプリ名 meta | **採用** |
| — | 速度3倍の追加 | 危険時1倍自動復帰など既存バランスへ影響。今回は**見送り** | 見送り |
| — | 売却の確認ダイアログ | 即時操作のテンポを優先。**見送り** | 見送り |
| — | ゲームバランス（HP/資金/波） | 開発者が v10 まで調整済み。**意図的に触らない** | 見送り |
| — | ズーム無効化(viewport) | ダブルタップ暴発対策と密結合。**触らない** | 見送り |

> 方針: **ゲーム挙動・難易度は変えない**。改善は「性能・UX・アクセシビリティ・メタ情報」に限定し、各変更は挙動非依存（behavior-preserving）であることを最優先する。

## 4. 実装内容（採用項目の具体方針）

1. **head にメタ情報追加**（F-3 / D-2）
   - `description`, `theme-color`, `apple-mobile-web-app-*`, `og:*`, `twitter:card`, 絵文字 favicon(data URI)。
2. **`prefers-reduced-motion` 対応**（Q-1）
   - `.pill.danger` の点滅とシートのトランジションを抑制。
3. **ルールモーダルの新設**（U-1 / D-1）
   - `#rulesModal` を `guideModal` 系クラスで追加。`btnRules` は `alert` から `openRulesModal()` へ。
4. **アイコンボタンに aria-label**（U-3）
   - `#btnPause`, `#btnSpeed`。
5. **毎フレーム DOM 書き込みの最適化**（F-1）
   - `setText()` / `setHtml()` ヘルパを追加し、`updateHud()` / `updateNotice()` を値変化時のみ書き込みに変更。
6. **ターゲットソートの事前計算**（F-2）
   - `dense` / `line` のスコアを `Map` に1回計算してからソート（タイブレーク `b.dist-a.dist` は維持）。
7. **2段階設置のトースト案内**（U-2）
   - `selectBuildType()` で「設置予定」状態に入ったとき、もう一度タップを促すトースト。

## 5. リスクと検証

- **挙動非依存**: 5・6 はソート順・表示文言を変えない（DOM 書き込み回数のみ削減）。7 は表示の追加のみ。3・4 は新規UI/属性。1・2 は head/CSS のみ。
- **検証**:
  - `node` の VM 構文チェックを改修後に再実行（PASS を確認）。
  - HTML 構造の目視（`<script>` ブロックは1つのまま、タグ対応）。
  - 主要導線の机上トレース: ホーム→名前→ゲーム（設置/強化/売却/狙い変更）→クリア/敗北→結果→共有/再戦。

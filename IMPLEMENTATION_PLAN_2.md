## 2. 課題と各ロールの修正案（出し合い）

### ① 建設プレビューのキャンセルが直感的でない
- 🧭: ユーザー案は「(A)同マス再タップでキャンセル」または「(B)明示的なキャンセルボタン」。
- ⚙️: 現仕様は **「同じマスをもう一度タップ＝設置確定」**（第1回でトースト「設置するマスをもう一度タップ」も追加済み）。(A) を採ると確定動作と衝突し、配置手段が失われる。→ **(B) を採用**。
- 🎨: 設置予定中はシートが閉じ盤面が見える状態。盤面下部中央に小さな「✕ 設置をやめる」フローティングボタンを出すのが最も邪魔にならない。
- 🔍: ボタンは `pointer-events` を持たせ、ボスバー等の他オーバーレイとz順で衝突させない。タップは既存のHUDボタン同様 `onclick` で十分（ダブルタップ抑止の対象外パスを確認済み）。

**決定**: 設置予定中だけ表示されるフローティング「✕ 設置をやめる」ボタンを追加。押下で `closeSheet(true)`（pendingBuild等をクリア）＋再描画＋トースト。表示/非表示は `render()` 内の同期関数で `game.pendingBuild` に追従。

### ② 貫通タワー（⚡）の射線が予測しづらい
- 🧭: ユーザー案どおり、テキスト補足で誤解を防ぐのが現実的。
- ⚙️: `TOWERS.pierce.detail` に一文追記すれば、**設置前の武器一覧・武器情報・図鑑のすべて**に一括反映でき、変更点が1か所で済む。
- 🎨/🔍: 文言は冒頭に置き「方向性のある武器」だと最初に伝わるようにする。

**決定**: `TOWERS.pierce.detail` の先頭に「標的に向かって直線の貫通ビームを撃つ。」を追記（設置前から視認可能）。

### ③ ボス（🐘・🐉）のHP可視性
- 🎨: HP10万級の「削れ感」を出すには、画面上部に大型の専用バー。残量で色を3段階（緑寄り→橙→赤）に変えると達成感が増す。
- ⚙️: レイアウトシフト（盤面の再リサイズ）を避けるため、`.stageWrap` 内の**絶対配置オーバーレイ（DOM）**にする。Canvas再描画ではなくDOMで描けば文字も鮮明。毎フレーム更新は `setText` ＋ 幅キャッシュで最小化。
- 🔍: バーは `pointer-events:none` にして盤面タップを透過。ボスが複数いる場合は最前（ゴールに最も近い＝最も危険）を表示。`reduced-motion` 時は幅トランジションを無効化。

**決定**: `enemy.boss === true` の敵が画面上にいる間だけ、`.stageWrap` 上部に大型ボスHPバー（名前・絵文字・実数/最大/％・残量3段階カラー）をオーバーレイ表示。`render()` から毎フレーム同期。

## 3. 実装詳細

1. **CSS 追加**: `.cancelBuild`（フローティング）／`.bossBar` 一式（ヘッダ・トラック・フィル・`data-tier` 配色）。`prefers-reduced-motion` に `.bossBarFill{transition:none}` を追加。
2. **HTML（`.stageWrap` 内）**: `#bossBar`（`#bossName` / `#bossHpText` / `#bossHpFill`）と `#btnCancelBuild` を追加。`#bossBar` は `aria-hidden`、`pointer-events:none`。
3. **データ**: `TOWERS.pierce.detail` の先頭に一文追記。
4. **JS**:
   - `syncBuildCancelBtn()` … `game.pendingBuild` に応じて表示トグル（変化時のみ）。
   - `syncBossBar()` … 最前ボスを探し、表示・名前・HP文・フィル幅・`data-tier` を更新（いずれも変化時のみ）。
   - `render()` 冒頭で上記2つを呼ぶ（`game` が無い時は両方とも非表示にする）。
   - `#btnCancelBuild` の `onclick`：`closeSheet(true); render(); toast("設置をやめました");`

## 4. リスクと検証

- **整合**: ②は文言追記のみで挙動不変。①は表示UIの追加＋既存 `closeSheet` の再利用で、設置確定フロー（同マス再タップ）は不変。③は描画オーバーレイの追加で、盤面リサイズ・タップ判定・ゲームロジックに非干渉（`pointer-events:none`）。
- **負荷**: 毎フレームのDOM更新は `setText`／幅・tierのキャッシュで、値変化時のみ。
- **検証**:
  - `node` VM 構文チェック PASS。
  - HTMLタグ対応バランス（div/section/button/style/script）。
  - 机上トレース: 設置予定→「✕設置をやめる」→他操作／第10・20波（🐘）・第30波（🐉）でボスバー表示・残量色・到達時の消滅／貫通の説明文が一覧・情報・図鑑に出ること。
- **見送り**: 同マス再タップでのキャンセル（確定動作と衝突）、貫通の方向プレビュー矢印（仕様上ターゲット未確定・過度な実装）。
